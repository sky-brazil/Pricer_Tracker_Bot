# International Readiness Checklist (GitHub + Upwork)

Use this checklist before sharing your repository with international clients.

## Language and presentation

- [ ] All client-facing files are in English.
- [ ] README explains business value, not only technical details.
- [ ] Portfolio examples are segmented by industry pain, not generic templates.
- [ ] Messaging sounds specific and human-curated.

## Technical credibility

- [ ] Core workflows have automated tests.
- [ ] CI validates baseline quality on push/PR.
- [ ] Environment setup is clear (`.env.example`, config templates).
- [ ] Errors and warnings are explicit and actionable.

## Commercial positioning

- [ ] Service packages include realistic scope and budget ranges.
- [ ] KPI targets are clearly stated for each offer.
- [ ] First milestone can show practical value quickly.
- [ ] Communication style fits discovery calls with non-technical stakeholders.

## Delivery trust signals

- [ ] Repository includes a software license.
- [ ] Commits are structured with descriptive messages.
- [ ] Portfolio links are easy to navigate.
- [ ] Documentation can be reused in proposals without extra rewriting.
